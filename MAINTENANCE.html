🚀 3. Cara Update Backend

⚠️ Setiap ada perubahan kode di src/, wajib rebuild bundle sebelum PM2 bisa menjalankan kode baru.

Alur Update (di mesin developer)

  1. Edit source code di src/
  2. Jalankan build + bundle: 
    npm run bundle:be
  3. Copy folder backend-runtime/ ke PC production (via USB / sharing)
  4. Di PC production, restart PM2: 
    pm2 restart backend

Hanya update .env (tanpa rebuild kode)

  1. Edit file backend-runtime/.env langsung di PC production
  2. Restart PM2: 
    pm2 restart backend

🔌 6. WebSocket

WebSocket URL: ws://<IP>:4001

Transport: websocket only (polling dinonaktifkan)

Topics yang tersedia

Topic (subscribe) Event (listen) Data Source
sequences sequences:update TB_R_SEQUENCE_BATTERY
summary summary:update TB_R_ANDON_GLOBAL
calls calls:update TB_R_ANDON_CALL
processes processes:update TB_R_POS_STATUS
downtime downtime:update TB_R_DOWNTIME_LOG

Cara pakai (dari Frontend)

const socket = io('http://192.168.1.167:4001', {
  transports: ['websocket'],
})

// Subscribe ke topic
socket.on('connect', () => {
  socket.emit('subscribe', 'summary')
})

// Terima data real-time
socket.on('summary:update', (data) => {
  console.log(data)
})

// Force refresh manual
socket.emit('sync', 'summary')

🔍 7. Troubleshooting

❌ Backend tidak bisa connect ke database

Log: ESOCKET atau Could not connect (sequence)
  1. Pastikan SQL Server service sedang berjalan
    services.msc → SQL Server (MSSQLSERVER) → Start
  2. Cek konfigurasi di backend-runtime/.env — pastikan MSSQL_SERVER, USER, PASSWORD benar
  3. Test koneksi manual: 
    sqlcmd -S localhost -U sa -P [password] -Q "SELECT 1"
  4. Backend akan auto-retry 5x dengan exponential backoff (1s → 2s → 4s → 8s → 16s)

❌ Port 4001 sudah dipakai

Log: EADDRINUSE — Port still in use 
# Cari proses di port 4001
netstat -ano | findstr :4001

# Kill proses (ganti XXXX dengan PID)
taskkill /PID XXXX /F

# Atau restart PM2
pm2 restart backend

❌ Frontend gagal connect WebSocket

  1. Pastikan URL WebSocket benar: ws://<IP-BE>:4001 bukan port FE
  2. Pastikan transport: ['websocket'] (bukan polling)
  3. Cek CORS — tambahkan IP FE ke ALLOWED_ORIGINS di .env
  4. Pastikan firewall Windows tidak block port 4001

❌ Data di Frontend tidak update setelah reconnect

Pastikan FE melakukan re-subscribe setelah reconnect 
// Di FE — pastikan subscribe ulang saat reconnect:
socket.on('connect', () => {
  socket.emit('subscribe', 'summary')
  socket.emit('subscribe', 'downtime')
  // atau force sync:
  socket.emit('sync', 'summary')
})

❌ PM2 tidak auto-start setelah PC restart

# Jalankan sekali saat setup awal
pm2 save
pm2 startup

# Cek apakah sudah tersimpan
pm2 list

❌ Swagger tidak bisa diakses

  1. Pastikan backend running: pm2 status backend
  2. Buka di browser: http://localhost:4001/api-docs
  3. Jika dari PC lain: http://192.168.1.167:4001/api-docs
